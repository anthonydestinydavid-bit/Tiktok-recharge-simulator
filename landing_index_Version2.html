A local/demo simulator for live-stream gift flows — mock wallets & recharge, gift catalog, real-time feed (Socket.IO), admin simulator to inject viewers/gifts, analytics, and webhook testing. Perfect for QA, demos, and UI/overlay development.

Start locally (recommended):

docker-compose up --build
# frontend: http://localhost:5173
# api: http://localhost:4000

Or run server and client separately (Node + Vite) — see README for development tips.

What you'll get

This simulator is delivered as a small full-stack project (Express + Socket.IO backend, React + TypeScript frontend). It provides realistic event flows for UI, overlays, webhook handling, and load-testing scenarios without touching real payment systems.

Use cases

  • Stream overlay development — listen to gift events and animate accordingly.
  • QA: validate handle of gift bursts, error cases, and latency.
  • Stakeholder demos — show gifting UX and revenue simulation.
  • Webhook consumers — test your endpoint receiving gift callbacks.

Next steps

  1. Run with Docker (recommended) or start server & client locally.
  2. Open the simulator, use Admin panel to simulate viewers/gifts.
  3. Use webhook-simulate to forward a gift.event to your endpoint for integration tests.
